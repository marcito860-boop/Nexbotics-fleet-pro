## Overview 500 Error Fixed

### Problem
Maintenance overview endpoint returning 500 Internal Server Error.

### Root Causes
1. `VehicleDowntimeModel.getStats()` used `duration_days` column (doesn't exist) - should use `duration_hours / 24`
2. SQL queries referenced `ms.title` - should be `ms.service_name`
3. `MaintenanceReminderModel` SQL queries had wrong column references

### Fixes
- Fixed `VehicleDowntimeModel.getStats()` to calculate days from hours
- Fixed `mapRowToDowntime()` to derive durationDays from durationHours
- Fixed all `ms.title` references to `ms.service_name` in reminder queries
- Fixed `next_service_mileage` to `next_service_km` in reminder generation SQL
