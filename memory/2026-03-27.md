## Comprehensive Bulk Import/Export System (NEW FEATURE)

### Overview
Full mass upload system for ALL data types in the fleet system, similar to the original fleet-system.

### Supported Import/Export Types

| Type | Description | Key Fields |
|------|-------------|------------|
| **vehicles** | Vehicle fleet | registration_number, make, model, year, fuel_type, status |
| **drivers** | Driver records | first_name, last_name, license_number, license_expiry |
| **inventory** | General inventory | sku, name, category, current_stock, unit_price |
| **maintenance_records** | Service history | vehicle_registration, service_type, cost, completed_date |
| **fuel_records** | Fuel fill-ups | vehicle_registration, date, liters, cost, odometer |
| **routes** | Trip/route data | vehicle_registration, route_date, route_name, actual_km, actual_fuel |
| **accidents** | Accident reports | vehicle_registration, accident_date, location, severity, damage_cost |
| **staff** | Staff/employees | staff_no, staff_name, email, department, role |
| **service_providers** | Garages/workshops | name, type, contact_person, phone, email |
| **spare_parts** | Parts inventory | part_number, name, category, quantity_in_stock, unit_cost |
| **maintenance_schedules** | PM schedules | vehicle_registration, schedule_type, interval_mileage, interval_months |

### API Endpoints

```bash
# Preview CSV before import (validate data)
POST /api/fleet/import/preview
Body: { "importType": "fuel_records", "csvContent": "..." }

# Start import job (async processing)
POST /api/fleet/import
Body: { "importType": "vehicles", "csvContent": "...", "fileName": "fleet.csv" }

# Check import status
GET /api/fleet/import/jobs/:jobId

# Download CSV template
GET /api/fleet/export/templates/:type
# Types: vehicles, drivers, inventory, maintenance_records, fuel_records,
#        routes, accidents, staff, service_providers, spare_parts, maintenance_schedules

# Export data to CSV
POST /api/fleet/export
Body: { "exportType": "maintenance_records", "format": "csv" }
```

### Smart Features

**Auto-Lookup by Reference:**
- Vehicle lookup by `registration_number` → links to vehicle_id
- Driver lookup by `driver_name` → links to driver_id
- Provider lookup by `provider_name` or `supplier_name` → links to provider_id

**Auto-Calculated Fields:**
- Fuel consumption rate (km/l) from actual_km / actual_fuel
- Next service date from interval_months
- Next service mileage from interval_mileage + current_mileage
- Warranty expiry from warranty_months + completed_date

**Auto-Updates:**
- Vehicle mileage updated on fuel records, routes, maintenance completion
- Vehicle last_service_date updated on maintenance completion

**Validation:**
- Required field checks
- Date format validation (YYYY-MM-DD)
- Numeric field validation
- Enum validation (service_type, schedule_type, etc.)
- Vehicle existence verification

## Automatic Migration Runner Added

**Time:** 5:45 AM (Asia/Shanghai)

### Problem
- Database migrations in `database/migrations/` weren't being applied automatically
- Render deploys don't automatically run SQL migrations
- Missing tables (`job_cards`, `trips`) causing 500 errors

### Solution
- Added `runMigrations()` function to `database/index.ts`
- Reads all `.sql` files from `database/migrations/` directory
- Tracks applied migrations in `schema_migrations` table
- Runs automatically on server startup before accepting requests
- Only applies new migrations (idempotent)

### Migration Flow
1. Server starts
2. Run migrations (creates/updates tables)
3. Start accepting requests
