## Requisition Module 400 Error Fix

**Issue:** Frontend sending requisition requests got 400 "Missing required fields" error.

**Root Cause:** Frontend doesn't include `requested_by` (staffId) in the request payload. Users authenticate but don't have a staff record linked.

**Solution:** Backend now auto-creates staff record when needed:
1. First tries to find existing staff by user's email
2. If not found, creates new staff record from user's JWT data
3. Uses that staff ID as the requisition requester

**Code Changes (requisitions.ts):**
```typescript
// Auto-create staff if user doesn't have one linked
let requesterId = normalizedData.requested_by;
if (!requesterId && req.user?.email) {
  // Find staff by email
  const staffByEmail = await query('SELECT id FROM staff WHERE email = $1', [req.user.email]);
  if (staffByEmail?.length > 0) {
    requesterId = staffByEmail[0].id;
  } else {
    // Create staff record
    const newStaffId = uuidv4();
    await query(`INSERT INTO staff (id, staff_name, email, role) VALUES (...)`, [...]);
    requesterId = newStaffId;
  }
}
```

**Commit:** `16323962`

### Key Lesson
When dealing with user/staff relationships, handle the case where a user exists but hasn't been linked to a staff record. Auto-creation provides better UX than forcing manual setup.

## Auto User Account Creation for Staff

**Feature:** When staff is added (single or bulk), user accounts are auto-created with temp passwords.

### Implementation Details

**Backend (`staff.ts`):**
- `POST /api/staff` - Single staff creation with optional user account
- `POST /api/staff/bulk-import` - CSV import with mass user creation
- Role mapping: Admin/Director/CEO → `admin`, Manager/Supervisor/Head → `manager`, others → `staff`
- Generates secure temp password via `generateSecurePassword()`
- Sets `must_change_password = true` in database

**Frontend (`StaffImportPage.tsx`):**
- CSV upload with drag & drop
- Preview first 5 rows before import
- Toggle for auto-creating user accounts
- Results dashboard showing temp passwords
- Download results CSV with all credentials

**First Login Flow:**
1. User logs in with email + temp password
2. Auto-redirected to `/change-password`
3. Must set new password (min 8 chars)
4. `must_change_password` set to false
5. Redirected to role-based dashboard

### Security Notes
- Temp passwords shown once to admin only
- Users cannot access system until password changed
- Password change enforced at middleware level
