# Memory Log - March 14, 2026

## Fleet Management System - Major Updates

### 1. Fixed Excel Imports (Critical)
- **Problem:** Routes import was a stub - did nothing
- **Fixed:** Implemented full routes import with:
  - Vehicle lookup by registration number
  - Driver lookup by name/staff number
  - Automatic consumption rate calculations
  - Variance tracking (actual vs target km)

### 2. Fixed Frontend API URL
- **Problem:** Frontend calling `/dashboard/stats` instead of `/api/dashboard/stats`
- **Fixed:** Hardcoded API URL to `https://fleet-api-0272.onrender.com/api`
- Deployed to Vercel: https://fleet-pro-omega.vercel.app/

### 3. Migrated to PostgreSQL (Critical)
- **Problem:** SQLite at `/tmp/fleet.db` was wiped on every Render restart
- **Solution:** Migrated to PostgreSQL for persistent storage
- **Files Changed:**
  - `backend/src/database/index.ts` - Now uses `pg` Pool
  - All routes updated from SQLite `?` to PostgreSQL `$1, $2...` syntax
  - Date functions: `date('now', '-X days')` → `CURRENT_DATE - INTERVAL 'X days'`

### 4. Added Admin Panel
- **Feature:** Reset Data button for pre-pilot cleanup
- **Location:** New ⚙️ Admin tab (admin only)
- **Function:** Clears vehicles, staff, routes, fuel, repairs - keeps admin users
- **Backend:** `/api/admin/reset-data` endpoint

### 5. Added Analytics Tab
- **Driver Productivity:**
  - Performance ranking by trips, efficiency, on-target %
  - Color-coded ratings (Excellent/Good/Needs Improvement)
  - Bar charts and pie charts
- **Maintenance Productivity:**
  - Garage analysis (repairs, hours, costs)
  - Technician ratings and efficiency scores
- **Optimization:** Added "Load Analytics" button (was auto-loading causing slowness)

### 6. Fixed PDF Export
- **Problem:** `jspdf-autotable` import wrong for ES modules
- **Fixed:** Changed from `import 'jspdf-autotable'` to `import autoTable from 'jspdf-autotable'`
