## 🔥 CRITICAL BUG DISCOVERED & FIXED

### Problem: Data Disappearing on Every Deploy
**Root Cause:** The database initialization code was dropping ALL TABLES on every server restart:
```typescript
// OLD DESTRUCTIVE CODE:
await pool.query(`
  DROP TABLE IF EXISTS requisitions CASCADE;
  DROP TABLE IF EXISTS repairs CASCADE;
  DROP TABLE IF EXISTS fuel_records CASCADE;
  ...etc
`);
```

**Impact:** Every time Render redeployed the backend (which happens frequently), ALL USER DATA was permanently deleted.

### Fix Applied:
- Removed all `DROP TABLE` statements from `backend/src/database/index.ts`
- Tables now only created if they don't exist (`CREATE TABLE IF NOT EXISTS`)
- Data will persist between deployments

### Reports Crash Fix:
- Added null safety to `Reports.tsx` fetchData() function
- Empty state handling for fuel records table
- Better error handling with Array.isArray() validation
- Fallback to empty arrays when API fails
